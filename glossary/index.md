# Glossary

<p></p><dl><dt><b><a name="SPTX7494374">adoption item</a></b></dt><a name="SPTX7494374">
<dd>A development work item that tracks when changes by one team must be adopted by another team.</dd>
</a></dl><a name="SPTX7494374">
<p></p></a><dl><a name="SPTX7494374"></a><dt><a name="SPTX7494374"><b></b></a><b><a name="SPTX3956523">agile</a></b></dt><a name="SPTX3956523">
</a><dd><a name="SPTX3956523">See </a><a href="#SPTX2907651">agile software development</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX2907651">agile software development (agile)</a></b></dt><a name="SPTX2907651">
<dd>A software development methodology that focuses on collaboration and
 quality to produce complete product increments at regular intervals, 
called iterations or sprints.</dd>
</a></dl><a name="SPTX2907651">
<p></p></a><dl><a name="SPTX2907651"></a><dt><a name="SPTX2907651"><b></b></a><b><a name="SPTX4281528">app</a></b></dt><a name="SPTX4281528">
<dd>A web or mobile device application.</dd>
</a></dl><a name="SPTX4281528">
<p></p></a><dl><a name="SPTX4281528"></a><dt><a name="SPTX4281528"><b></b></a><b><a name="SPTX2262995">artifact</a></b></dt><a name="SPTX2262995">
</a><dd><a name="SPTX2262995">An entity that is used or produced by a 
software development process. Examples of artifacts are models, source 
files, scripts, and binary executable files. See also </a><a href="#SPTX2017871">component</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX7344035">auto-complete</a></b></dt><a name="SPTX7344035">
</a><dd><a name="SPTX7344035">See </a><a href="#SPTX7344030">code completion</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX7494379">auto-deploy</a></b></dt><a name="SPTX7494379">
<dd>(1) An automated process that retrieves and packages build output, 
and then moves that output to another location so that users can run or 
test it.</dd>
<dd>(2) To automatically retrieve and package build output, then move that output to another location for users to run or test it.</dd>
</a></dl><a name="SPTX7494379">
<p></p></a><dl><a name="SPTX7494379"></a><dt><a name="SPTX7494379"><b></b></a><b><a name="SPTX2315215">backlog</a></b></dt><a name="SPTX2315215">
<dd>In agile development, a list of features or technical tasks that are
 necessary for a project or release that the development team assesses 
and prioritizes to assign work.</dd>
</a></dl><a name="SPTX2315215">
<p></p></a><dl><a name="SPTX2315215"></a><dt><a name="SPTX2315215"><b></b></a><b><a name="SPTX7233930">boilerplate</a></b></dt><a name="SPTX7233930">
<dd>A container for one application and its associated runtime environment and predefined services for a particular domain.</dd>
</a></dl><a name="SPTX7233930">
<p></p></a><dl><a name="SPTX7233930"></a><dt><a name="SPTX7233930"><b></b></a><b><a name="SPTX2050278">branch</a></b></dt><a name="SPTX2050278">
<dd>(1) To create a stream for parallel or insulated development.</dd>
<dd>(2) A stream for insulated or parallel development.</dd>
</a></dl><a name="SPTX2050278">
<p></p></a><dl><a name="SPTX2050278"></a><dt><a name="SPTX2050278"><b></b></a><b><a name="SPTX2197543">builder</a></b></dt><a name="SPTX2197543">
<dd>A tool that builds and compiles code.</dd>
</a></dl><a name="SPTX2197543">
<p></p></a><dl><a name="SPTX2197543"></a><dt><a name="SPTX2197543"><b></b></a><b><a name="SPTX2050789">CD</a></b></dt><a name="SPTX2050789">
</a><dd><a name="SPTX2050789">See </a><a href="#SPTX7233913">continuous delivery</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX2255031">change set</a></b></dt><a name="SPTX2255031">
<dd>A group of related changes to files, folders, or symbolic links in a workspace or stream.</dd>
</a></dl><a name="SPTX2255031">
<p></p></a><dl><a name="SPTX2255031"></a><dt><a name="SPTX2255031"><b></b></a><b><a name="SPTX2008863">CLI</a></b></dt><a name="SPTX2008863">
</a><dd><a name="SPTX2008863">See </a><a href="#SPTX2051424">command-line interface</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX2806164">clone</a></b></dt><a name="SPTX2806164">
<dd>(1) In Git, a local copy of a repository, which can be edited 
offline, that is connected to and can be synchronized with the original 
remote version.</dd>
<dd>(2) In Git, to create a local copy of a repository, which can be 
edited offline, that is connected to and can be synchronized with the 
original remote version.</dd>
</a></dl><a name="SPTX2806164">
<p></p></a><dl><a name="SPTX2806164"></a><dt><a name="SPTX2806164"><b></b></a><b><a name="SPTX7344030">code completion</a></b></dt><a name="SPTX7344030">
<dd>A feature of many IDEs and text editors that predictively completes 
content (words, phrases, tags, and so on) while the user types.</dd>
</a></dl><a name="SPTX7344030">
<p></p></a><dl><a name="SPTX7344030"></a><dt><a name="SPTX7344030"><b></b></a><b><a name="SPTX2051424">command-line interface (CLI)</a></b></dt><a name="SPTX2051424">
<dd>A computer interface in which the input and output are text based.</dd>
</a></dl><a name="SPTX2051424">
<p></p></a><dl><a name="SPTX2051424"></a><dt><a name="SPTX2051424"><b></b></a><b><a name="SPTX2000758">commit</a></b></dt><a name="SPTX2000758">
<dd>(1) To store a snapshot of the current state of a project.</dd>
<dd>(2) A snapshot of the current state of a project in a repository. A project's history is defined by a series of commits.</dd>
</a></dl><a name="SPTX2000758">
<p></p></a><dl><a name="SPTX2000758"></a><dt><a name="SPTX2000758"><b></b></a><b><a name="SPTX2017871">component</a></b></dt><a name="SPTX2017871">
</a><dd><a name="SPTX2017871">A unit of organization consisting of a reusable set of engineering artifacts. See also </a><a href="#SPTX2262995">artifact</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX7233913">continuous delivery (CD)</a></b></dt><a name="SPTX7233913">
<dd>A software development practice that employs techniques such as 
continuous testing, continuous integration, and continuous deployment so
 that new features and fixes are packaged and deployed rapidly and at 
low risk to test environments and then to customers.</dd>
</a></dl><a name="SPTX7233913">
<p></p></a><dl><a name="SPTX7233913"></a><dt><a name="SPTX7233913"><b></b></a><b><a name="SPTX2363941">dashboard</a></b></dt><a name="SPTX2363941">
<dd>A web user interface component that provides information about the 
project status at a glance. Users can view project summary information 
or more detailed information.</dd>
</a></dl><a name="SPTX2363941">
<p></p></a><dl><a name="SPTX2363941"></a><dt><a name="SPTX2363941"><b></b></a><b><a name="SPTX2117804">defect</a></b></dt><a name="SPTX2117804">
<dd>A project element that allows stakeholders to track bugs and bug fixes.</dd>
</a></dl><a name="SPTX2117804">
<p></p></a><dl><a name="SPTX2117804"></a><dt><a name="SPTX2117804"><b></b></a><b><a name="SPTX2104544">deployment</a></b></dt><a name="SPTX2104544">
<dd>A multistage process that retrieves the output of a build, packages 
the output, and moves the package to another location so it can be 
tested or run.</dd>
</a></dl><a name="SPTX2104544">
<p></p></a><dl><a name="SPTX2104544"></a><dt><a name="SPTX2104544"><b></b></a><b><a name="SPTX5784896">DevOps</a></b></dt><a name="SPTX5784896">
<dd>A software methodology that integrates application development and IT operations.</dd>
</a></dl><a name="SPTX5784896">
<p></p></a><dl><a name="SPTX5784896"></a><dt><a name="SPTX5784896"><b></b></a><b><a name="SPTX5788994">epic</a></b></dt><a name="SPTX5788994">
<dd>In agile development, a work item that defines a big use case with a
 large or unknown amount of work, which is often completed over several 
sprints. Epics can be divided into stories.</dd>
</a></dl><a name="SPTX5788994">
<p></p></a><dl><a name="SPTX5788994"></a><dt><a name="SPTX5788994"><b></b></a><b><a name="SPTX2022601">fetch</a></b></dt><a name="SPTX2022601">
<dd>In Git, to retrieve changes from a repository without merging them.</dd>
</a></dl><a name="SPTX2022601">
<p></p></a><dl><a name="SPTX2022601"></a><dt><a name="SPTX2022601"><b></b></a><b><a name="SPTX2023301">fork</a></b></dt><a name="SPTX2023301">
<dd>(1) In Git, to create a divergent copy of a repository that can be 
modified without affecting or having to synchronize with the original.</dd>
<dd>(2) In Git, a divergent copy of a repository that can be modified without affecting or having to synchronize with the original.</dd>
</a></dl><a name="SPTX2023301">
<p></p></a><dl><a name="SPTX2023301"></a><dt><a name="SPTX2023301"><b></b></a><b><a name="SPTX7494416">Git</a></b></dt><a name="SPTX7494416">
<dd>An open source program for source control management.</dd>
</a></dl><a name="SPTX7494416">
<p></p></a><dl><a name="SPTX7494416"></a><dt><a name="SPTX7494416"><b></b></a><b><a name="SPTX4585337">IaaS</a></b></dt><a name="SPTX4585337">
</a><dd><a name="SPTX4585337">See </a><a href="#SPTX4585332">infrastructure as a service</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX7301758">IBM Bluemix</a></b></dt><a name="SPTX7301758">
<dd>An open-standards, cloud-based platform for building, managing, and 
running apps of all types, such as web, mobile, big data, and smart 
devices. Capabilities include Java, mobile back-end development, and 
application monitoring, as well as features from ecosystem partners and 
open sourceâ€”all provided as-a-service in the cloud.</dd>
</a></dl><a name="SPTX7301758">
<p></p></a><dl><a name="SPTX7301758"></a><dt><a name="SPTX7301758"><b></b></a><b><a name="SPTX2115755">IDE</a></b></dt><a name="SPTX2115755">
</a><dd><a name="SPTX2115755">See </a><a href="#SPTX2116934">integrated development environment</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX4015600">impediment</a></b></dt><a name="SPTX4015600">
<dd>An item that prevents progress or a work item to track something that prevents progress.</dd>
</a></dl><a name="SPTX4015600">
<p></p></a><dl><a name="SPTX4015600"></a><dt><a name="SPTX4015600"><b></b></a><b><a name="SPTX4585332">infrastructure as a service (IaaS)</a></b></dt><a name="SPTX4585332">
<dd>The delivery of a computer infrastructure, including server 
functionality, networking functionality, data center functionality, and 
storage functionality as an outsourced service.</dd>
</a></dl><a name="SPTX4585332">
<p></p></a><dl><a name="SPTX4585332"></a><dt><a name="SPTX4585332"><b></b></a><b><a name="SPTX2116934">integrated development environment (IDE)</a></b></dt><a name="SPTX2116934">
<dd>A set of software development tools, such as source editors, 
compilers, and debuggers, that are accessible from a single user 
interface.</dd>
</a></dl><a name="SPTX2116934">
<p></p></a><dl><a name="SPTX2116934"></a><dt><a name="SPTX2116934"><b></b></a><b><a name="SPTX2026316">iteration</a></b></dt><a name="SPTX2026316">
</a><dd><a name="SPTX2026316">See </a><a href="#SPTX6248609">sprint</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX2858069">manifest file</a></b></dt><a name="SPTX2858069">
<dd>A file that contains information, such as settings, instructions, or metadata, about other files or artifacts.</dd>
</a></dl><a name="SPTX2858069">
<p></p></a><dl><a name="SPTX2858069"></a><dt><a name="SPTX2858069"><b></b></a><b><a name="SPTX2030107">merge</a></b></dt><a name="SPTX2030107">
<dd>(1) The commit that results after bringing content from another branch into the current branch.</dd>
<dd>(2) To fetch content from one branch and apply it to another branch.</dd>
</a></dl><a name="SPTX2030107">
<p></p></a><dl><a name="SPTX2030107"></a><dt><a name="SPTX2030107"><b></b></a><b><a name="SPTX2118203">milestone</a></b></dt><a name="SPTX2118203">
<dd>A significant event in a project or process that is used to determine progress toward goals.</dd>
</a></dl><a name="SPTX2118203">
<p></p></a><dl><a name="SPTX2118203"></a><dt><a name="SPTX2118203"><b></b></a><b><a name="SPTX2062278">open source</a></b></dt><a name="SPTX2062278">
<dd>Pertaining to software whose source code is publicly available for 
use or modification. Open source software is typically developed as a 
public collaboration and made freely available, although its use and 
redistribution might be subject to licensing restrictions. Linux is a 
well known example of open source software.</dd>
</a></dl><a name="SPTX2062278">
<p></p></a><dl><a name="SPTX2062278"></a><dt><a name="SPTX2062278"><b></b></a><b><a name="SPTX2029790">PaaS</a></b></dt><a name="SPTX2029790">
</a><dd><a name="SPTX2029790">See </a><a href="#SPTX2029786">platform as a service</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX2029786">platform as a service (PaaS)</a></b></dt><a name="SPTX2029786">
<dd>The delivery of a computing platform, including applications, 
optimized middleware, development tools, and Java and Web 2.0 runtime 
environments, as a service over the Internet.</dd>
</a></dl><a name="SPTX2029786">
<p></p></a><dl><a name="SPTX2029786"></a><dt><a name="SPTX2029786"><b></b></a><b><a name="SPTX2090430">priority</a></b></dt><a name="SPTX2090430">
<dd>In software development, a rank assigned to requirements or work items to determine the order in which they are worked on.</dd>
</a></dl><a name="SPTX2090430">
<p></p></a><dl><a name="SPTX2090430"></a><dt><a name="SPTX2090430"><b></b></a><b><a name="SPTX2035151">project</a></b></dt><a name="SPTX2035151">
<dd>An organized collection used to group folders or packages. Projects 
are used for building, version management, sharing, and organizing 
resources related to a single work effort.</dd>
</a></dl><a name="SPTX2035151">
<p></p></a><dl><a name="SPTX2035151"></a><dt><a name="SPTX2035151"><b></b></a><b><a name="SPTX2064205">pull</a></b></dt><a name="SPTX2064205">
<dd>In Git, to fetch and merge changes at the same time.</dd>
</a></dl><a name="SPTX2064205">
<p></p></a><dl><a name="SPTX2064205"></a><dt><a name="SPTX2064205"><b></b></a><b><a name="SPTX2035465">push</a></b></dt><a name="SPTX2035465">
<dd>In Git, to send local, committed changes to a remote repository so that others can access them.</dd>
</a></dl><a name="SPTX2035465">
<p></p></a><dl><a name="SPTX2035465"></a><dt><a name="SPTX2035465"><b></b></a><b><a name="SPTX7494186">repo</a></b></dt><a name="SPTX7494186">
</a><dd><a name="SPTX7494186">See </a><a href="#SPTX2036865">repository</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX2036865">repository (repo)</a></b></dt><a name="SPTX2036865">
<dd>A persistent storage area for data and other application resources.</dd>
</a></dl><a name="SPTX2036865">
<p></p></a><dl><a name="SPTX2036865"></a><dt><a name="SPTX2036865"><b></b></a><b><a name="SPTX3889804">repository workspace</a></b></dt><a name="SPTX3889804">
</a><dd><a name="SPTX3889804">A repository object that includes one or 
more components. Repository workspaces are typically used by individual 
team members to contain their changes in progress. Team members deliver 
their changes from their repository workspace to the stream and accept 
changes from other team members into their repository workspace from the
 stream. Every repository workspace has an owner, and only the owner can
 make changes in the workspace. See also </a><a href="#SPTX2096037">workspace</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX7494440">retrospective</a></b></dt><a name="SPTX7494440">
<dd>In agile development, a meeting or a work item that evaluates successes and needed improvements after a sprint.</dd>
</a></dl><a name="SPTX7494440">
<p></p></a><dl><a name="SPTX7494440"></a><dt><a name="SPTX7494440"><b></b></a><b><a name="SPTX2037566">runtime environment</a></b></dt><a name="SPTX2037566">
<dd>A set of resources that are used to run a program or process.</dd>
</a></dl><a name="SPTX2037566">
<p></p></a><dl><a name="SPTX2037566"></a><dt><a name="SPTX2037566"><b></b></a><b><a name="SPTX4585391">SaaS</a></b></dt><a name="SPTX4585391">
</a><dd><a name="SPTX4585391">See </a><a href="#SPTX4585386">software as a service</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX2116206">SCM</a></b></dt><a name="SPTX2116206">
</a><dd><a name="SPTX2116206">See </a><a href="#SPTX2116320">software configuration management</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX7474016">Scrum</a></b></dt><a name="SPTX7474016">
<dd>An agile software programming method that uses small, 
self-organizing, cross-functional teams, and iterative, incremental 
practices.</dd>
</a></dl><a name="SPTX7474016">
<p></p></a><dl><a name="SPTX7474016"></a><dt><a name="SPTX7474016"><b></b></a><b><a name="SPTX3266541">scrum</a></b></dt><a name="SPTX3266541">
<dd>A daily, informal meeting between stakeholders in an agile software 
development project. Participants must include a scrum master, the 
product owner, and the team and might include the product manager and 
other stakeholders.</dd>
</a></dl><a name="SPTX3266541">
<p></p></a><dl><a name="SPTX3266541"></a><dt><a name="SPTX3266541"><b></b></a><b><a name="SPTX2038343">service</a></b></dt><a name="SPTX2038343">
<dd>A cloud extension that provides ready-for-use functionality, such as
 database, messaging, and web software for running code. Services 
usually do not require installation or maintenance and can be combined 
to create applications.</dd>
</a></dl><a name="SPTX2038343">
<p></p></a><dl><a name="SPTX2038343"></a><dt><a name="SPTX2038343"><b></b></a><b><a name="SPTX2375015">severity</a></b></dt><a name="SPTX2375015">
<dd>In software development, a measurement of the importance for a unit of work, for example, critical, high, medium, or low.</dd>
</a></dl><a name="SPTX2375015">
<p></p></a><dl><a name="SPTX2375015"></a><dt><a name="SPTX2375015"><b></b></a><b><a name="SPTX4585386">software as a service (SaaS)</a></b></dt><a name="SPTX4585386">
<dd>A model of software deployment whereby software including business 
processes, enterprise applications, and collaboration tools, are 
provided as a service to customers.</dd>
</a></dl><a name="SPTX4585386">
<p></p></a><dl><a name="SPTX4585386"></a><dt><a name="SPTX4585386"><b></b></a><b><a name="SPTX2116320">software configuration management (SCM)</a></b></dt><a name="SPTX2116320">
<dd>The tracking and control of software development. SCM systems typically offer version control and team programming features.</dd>
</a></dl><a name="SPTX2116320">
<p></p></a><dl><a name="SPTX2116320"></a><dt><a name="SPTX2116320"><b></b></a><b><a name="SPTX2039442">space</a></b></dt><a name="SPTX2039442">
<dd>An organizational unit that contains applications and services in 
the Cloud Foundry infrastructure and can be used to store and track 
application resources.</dd>
</a></dl><a name="SPTX2039442">
<p></p></a><dl><a name="SPTX2039442"></a><dt><a name="SPTX2039442"><b></b></a><b><a name="SPTX6248609">sprint</a></b></dt><a name="SPTX6248609">
<dd>In Scrum development, a set period of time in which work is 
completed. Sprints can vary in length, for example, from 1 to 4 weeks, 
but typically have a fixed duration within a project.</dd>
</a></dl><a name="SPTX6248609">
<p></p></a><dl><a name="SPTX6248609"></a><dt><a name="SPTX6248609"><b></b></a><b><a name="SPTX2067189">stage</a></b></dt><a name="SPTX2067189">
<dd>To mark a set of files as being ready for commit.</dd>
</a></dl><a name="SPTX2067189">
<p></p></a><dl><a name="SPTX2067189"></a><dt><a name="SPTX2067189"><b></b></a><b><a name="SPTX5027530">story</a></b></dt><a name="SPTX5027530">
<dd>A development work item that defines part of a use case or a specific contribution to the value of the overall product.</dd>
</a></dl><a name="SPTX5027530">
<p></p></a><dl><a name="SPTX5027530"></a><dt><a name="SPTX5027530"><b></b></a><b><a name="SPTX7344042">syntax checking</a></b></dt><a name="SPTX7344042">
<dd>A feature that parses code and attempts to interpret it before 
compilation in order to discover syntax errors that might cause apps to 
not compile properly, then notifies the user.</dd>
</a></dl><a name="SPTX7344042">
<p></p></a><dl><a name="SPTX7344042"></a><dt><a name="SPTX7344042"><b></b></a><b><a name="SPTX2007852">task</a></b></dt><a name="SPTX2007852">
<dd>In agile development, a work item that defines a specific piece of work.</dd>
</a></dl><a name="SPTX2007852">
<p></p></a><dl><a name="SPTX2007852"></a><dt><a name="SPTX2007852"><b></b></a><b><a name="SPTX2092493">tile</a></b></dt><a name="SPTX2092493">
<dd>A visual representation of a running application that provides status on a dashboard.</dd>
</a></dl><a name="SPTX2092493">
<p></p></a><dl><a name="SPTX2092493"></a><dt><a name="SPTX2092493"><b></b></a><b><a name="SPTX2312511">timeline</a></b></dt><a name="SPTX2312511">
<dd>An area of activity in a project that typically has its own schedule, deliverables, teams, and process.</dd>
</a></dl><a name="SPTX2312511">
<p></p></a><dl><a name="SPTX2312511"></a><dt><a name="SPTX2312511"><b></b></a><b><a name="SPTX7494456">track build item</a></b></dt><a name="SPTX7494456">
<dd>A development work item that tracks required fixes after a failed build.</dd>
</a></dl><a name="SPTX7494456">
<p></p></a><dl><a name="SPTX7494456"></a><dt><a name="SPTX7494456"><b></b></a><b><a name="SPTX7290116">user sentiment</a></b></dt><a name="SPTX7290116">
<dd>A user-rating score that is derived from gathering users' 
application ratings from the major application stores. The function 
crawls all ratings and reviews to generate a cumulative mobile app score
 that combines app quality, user satisfaction, and the overall amount of
 positive and negative feedback that users provide.</dd>
</a></dl><a name="SPTX7290116">
<p></p></a><dl><a name="SPTX7290116"></a><dt><a name="SPTX7290116"><b></b></a><b><a name="SPTX2844389">WAR</a></b></dt><a name="SPTX2844389">
</a><dd><a name="SPTX2844389">See </a><a href="#SPTX2116506">web archive</a>.</dd>
</dl>
<p></p><dl><dt><b><a name="SPTX2116506">web archive (WAR)</a></b></dt><a name="SPTX2116506">
<dd>A compressed file format, defined by the Java EE standard, for 
storing all the resources required to install and run a web application
